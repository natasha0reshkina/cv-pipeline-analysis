# Данные

Полные наборы изображений не включены в репозиторий, потому что их объем слишком велик для хранения вместе с кодом и результатами экспериментов. Все использованные изображения вынесены в отдельное хранилище и доступны по ссылке

https://drive.google.com/drive/folders/1kvCNNANxSwgO_agvzSW8O-SmMULrDRp9?usp=sharing

Подготовка данных была отдельной и довольно большой частью работы. Базовый набор clean я собирала из валидационной части open images v7. Для этого в ноутбуке загружалась выборка изображений, после чего подходящие файлы копировались в отдельную папку clean, которая затем использовалась как основной спокойный сценарий для экспериментов

Набор high res я формировала отдельно. Сначала я пробовала отбирать крупные изображения из уже собранного clean, а затем расширила пул данных и загрузила большую выборку из open images v7, чтобы искать действительно тяжелые по размеру примеры. После этого изображения сортировались по размеру, и в итоговый набор попадали двести самых крупных файлов. Именно этот набор использовался для анализа поведения пайплайна на входах высокого разрешения

Для проверки устойчивости я отдельно строила наборы с искусственными искажениями. В качестве основы использовались изображения из clean, для которых затем генерировались версии с размытием, шумом и jpeg сжатием на нескольких уровнях. Это было нужно для того, чтобы оценить, как пайплайн ведет себя не только на обычных данных, но и в более сложных условиях

Папка hard в ноутбуке уже подключается как готовый набор более трудных случаев. При этом сам ноутбук показывает работу с этой папкой в экспериментах, но не содержит отдельного автоматического блока, в котором этот набор полностью собирается с нуля. Поэтому для описания репозитория корректно считать, что clean и high res были собраны непосредственно в ноутбуке, а hard использовался как заранее подготовленный набор сложных примеров

Таким образом, данные в проекте не были взяты как одна готовая папка. Для исследования отдельно подбирались базовые изображения, изображения высокого разрешения и наборы с искажениями, чтобы можно было сравнивать производительность, качество и устойчивость пайплайна в разных сценариях
